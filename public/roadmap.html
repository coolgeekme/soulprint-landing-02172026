SoulPrint

Product Roadmap — Path to App Store

January 2026

1. Objective

The primary objective is to get SoulPrint approved and listed on the Apple App Store. This document outlines what has been built, what remains to be completed, and the specific requirements for App Store submission.

2. Current Status

SoulPrint's core functionality is complete and deployed. The web application is live and operational with the following capabilities:

2.1 Authentication

User authentication is fully implemented using Supabase Auth. Users can sign up and log in via email/password or Google OAuth. Session management and secure token handling are in place.

2.2 Data Import

The ChatGPT data import system is operational. Users can upload their ChatGPT export ZIP file, which is processed entirely client-side in the browser. This privacy-first approach ensures that raw conversation data never leaves the user's device. Only the extracted profile data (interests, communication style, embeddings) is transmitted to our servers.

2.3 Memory System

The infinite memory architecture is implemented and functional. Conversation chunks are embedded as vectors and stored in PostgreSQL using the pgvector extension. When users send messages, the system performs semantic search to retrieve relevant memories, which are then injected into the AI's context for personalized responses.

2.4 Chat Interface

The chat interface supports streaming responses, AI naming/personalization, and message history persistence. Basic mobile support exists—the interface functions on mobile devices but is not yet optimized for the native mobile experience required by Apple.

2.5 Deployment

The application is deployed on Vercel with the database hosted on Supabase. The system is production-ready and serving users.

3. Requirements for App Store Submission

To submit SoulPrint to the Apple App Store, the following items must be completed:

3.1 Native Application Wrapper

Apple requires native iOS applications for App Store distribution. The web application must be wrapped using a framework such as Expo or React Native to create a native iOS build. This wrapper will enable distribution through the App Store while maintaining the existing web-based functionality.

Specific deliverables:

  • Expo or React Native project configuration
  • iOS build configuration with proper signing certificates
  • Provisioning profiles for development and distribution
  • TestFlight build for internal testing before submission

3.2 App Store Assets

Apple has specific requirements for visual assets that must accompany every App Store listing:

AssetSpecification
App Icon1024×1024 pixels, PNG format, no transparency
Screenshots (6.7" display)1290×2796 pixels, minimum 3 images
Screenshots (5.5" display)1242×2208 pixels, minimum 3 images
App Preview (optional)Video showcasing app functionality

Additionally, compelling marketing copy is required:

  • App name (30 characters max)
  • Subtitle (30 characters max)
  • Description (4000 characters max)
  • Keywords (100 characters max)
  • What's New text for updates

3.3 Legal Requirements

Apple mandates specific legal documentation and functionality:

Privacy Policy: A publicly accessible privacy policy page that clearly discloses what data is collected, how it is used, and how users can request deletion. This is especially critical given SoulPrint's handling of conversation data.

Terms of Service: A user agreement outlining the terms of using the application.

Account Deletion: Per Apple's guidelines, users must have the ability to delete their account and associated data from within the app. This functionality must be easily discoverable.

3.4 Apple Developer Program

Submission requires enrollment in the Apple Developer Program:

  • Apple Developer account ($99/year)
  • App Store Connect access for managing the listing
  • Completed app information and pricing configuration
  • Age rating questionnaire
  • Export compliance information

4. Submission Process

Once all requirements are met, the submission process follows these steps:

  1. Build Upload: Upload the iOS build to App Store Connect via Xcode or Transporter
  2. Metadata Entry: Complete all required fields in App Store Connect
  3. Review Notes: Provide demo account credentials and any special instructions for Apple reviewers
  4. Submit for Review: Submit the app for Apple's review process
  5. Review Period: Apple reviews approximately 90% of submissions within 24 hours
  6. Approval or Feedback: Address any rejection reasons if applicable and resubmit

5. Timeline Estimate

Development work remaining: 3-5 days

Apple review process: 1-7 days (first submissions may take longer)

Total estimated time to App Store: 1-2 weeks

6. Post-Launch Considerations

After initial App Store approval, future development priorities include:

  • Android app (Google Play Store)
  • Push notifications for engagement
  • Additional import sources (WhatsApp, Telegram)
  • UI/UX improvements based on user feedback
  • Performance optimization
